HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • HTML attribute defines additional info about an element and is always in the beginning
  • metadata is data about the webpage that browser uses to effectively serve webpage to users
  • Elements nested inside other elements are child elements, parent element holds them all
  • A tag refers only to what appears inside angle brackets, while an element is from opening to closing tag
  • h1-h6 are headings, start with h1
  • p = paragraph or block of text
  • ul = unordered list, ol = ordered list, li = list items
  • section elements are semantic elements that define a section in a doc that contains thematic content
  • br = line break, a = anchor, a link to webpage and href is an attribute that points to URL for the link

CSS

  • A margin indicates how much space we want around the outside of an element
  • A padding inidicates how much space we want around the content inside an element
  • Incline CSS: used to style an element directly in HTML file, to specific tag. Useful for quick and permanent changes, but less flexible
  • Internal Style Sheet: HTML directly using style element in the head element
  • External CSS style sheet: most common option and allows rules in separate file to make design changes easier. Link said file using link element in HTML head
  • CSS Rule-Selector: defines element or attributes which rules or declarations will apply
  • CSS RUle-declarations: CSS property, such as color, and the value of that property, ex.blue
  • *: selector to make a rule for all elements on page
  • Class attributes: a way for CSS declarations to add styles to only certain elements
  • Use period before class attribute to designate it as a class selector

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch name: creates a new branch and switches to it
  • git commit -m: commit staged changes to repository, the -m flag is to associate a message with our commit
  • git pull origin main: pull in the base branch
  • git push origin *insert feature branch here*: push up all of our changes
  • -A flag indicates we want to add all changes
  • ls: list of directories and files
  • cd: change directory cd ..: go back
  • pwd: print working directory
  • mkdir: make directory
  • touch: new file
  • start: open file

JavaScript

  • A variable is a named container that allows us to store data in our code
  • Control flow is the order in which a computer executes code in a script
  • Const keyword allows us to create variables that can't be reassigned value
  • let keyword allows us to reassign values
  • console.log method allows us to output message to web console by adding an argument
  • Right click and select inspect, then select console tab to see console/log method in action
  • script.js is linked within body element of HTML
  • If Statement: Code will only execute if statement meets a condition or is true
  • Falsy = 0, -0, "", null, undefined, NaN, Boolean false. Truthy = everything else
  • Array is a single variable that is used to hold a group of data, typically data is related in some way, but can hold different types of data
  • Array Index is the location of data inside an array with a unique number to access inidividual data points. In JavaScript zero-indexed and sequential
  • A for loop uses sequential pattern of indices to perform task on all items in array, this allows single code block to be executed over and over
  • A for loop has 3 important statements: starting point of loop ex)let x = 0; condition statement x being less than array length ex) x < shapes.length; and the final statement which allows it to iterate and increase by one ex)x++
  • A function is a set of intructions that tells the computer how to perform a certain task
  • Functions do not automatically execute, must call a function by name to do so
  • To declare a function write function, then give it a unique name and include paranthesis right after, within curly brackets write code that should be run
